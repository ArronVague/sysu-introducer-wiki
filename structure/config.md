# 配置文件

本系统会调用很多外部的 API 或服务，
这些代码秘钥和参数不能硬编码进入代码中，
因此需要使用配置文件统一管理

## 文件结构

配置信息是对每个模块的具体信息和配置。
配置信息是对象的嵌套，
其中第一层对应各个模块，下一层则是不同实现类型对应的配信息。

```typescript
interface Config {
    name: Map<str, Map<str, number | string | boolean>>;
}
```

> 注意：配置文件中属性的类型

## 生命周期

每个模块在启动前都会重新加载配置信息，
也就是说模块在运行过程中配置信息是不会改变的。

> 在目前的实现中，每次启动时只会从内存中的副本中进行读取，
> 而不是本地的内存文件。
> 但是通过程序修改配置信息之后，系统会将配置信息重新写会。
